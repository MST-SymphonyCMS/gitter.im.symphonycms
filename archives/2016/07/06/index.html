Code is king. Code is the best documentation you can get. Code gets results.

But code still needs a shared direction :)

I see interesting stuff happening on Github and it feels like you and Brendan really want to push things forward, to modernise the code base. This is cool! But I’m still trying to figure out the bigger picture. You see, it has been noted a few times, that we all seem to head in the same direction – nevertheless all the concepts that have been posted here focus on very different things: core modularisation (code structure), separation of concerns (project structure), flexibility and universality (multiple workspaces).

And already by now, these concepts (even the Gists) are buried in this endless stream of post on Gitter.
So what’s a better place to keep and develop these ideas? The Symphony blog? Medium?

Communication is an absolute necessity to make an open source project a success. That's my biggest problem, since I suck at it. I try to make the best of what I can do. I am really open to let anybody do things on the communication side of things ;)

If this is a working open source project, it shouldn’t be up to you to handle everything. Public communication can and maybe should be handled by someone else: still there needs to be place where you jot down your ideas and plans. Where could this be? In this chat?
