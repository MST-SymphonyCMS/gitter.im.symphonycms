"Passing server variables through the environment was cool in 1999 when CGI was a thing, and web servers forked on every request and then forked again to run the CGI.

"Of the two easiest (and therefore most common) ways of using PHP, one (Apache+mod_php) still only works with the one-process=one-request model so it populates the environment just like CGI used to do, and the other (FastCGI) employs a compatibility shim like I described above. (Basically it has a custom getenv that silently diverts to the list of server variables.)
